copy功能实现

1.复制功能

经常遇到拷贝复制功能的开发，在项目中总结了现有拷贝功能的常见实现方法，并对其优缺点进行比较分析，便于今后开发中拷贝功能的快速实现。

2.实现方法及原理

现在常见的方法主要是以下两种：
第三方库：clipboard.js
原生方法：document.execCommand()

1. document.execCommand

MDN 上定义：

which allows one to run commands to manipulate the contents of the editable region.

当一个HTML文档切换到设计模式时，document暴露 execCommand 方法，该方法允许运行命令来操纵可编辑内容区域的元素。
bool = document.execCommand(aCommandName, aShowDefaultUI, aValueArgument)

方法返回一个 Boolean 值，表示操作是否成功。
aCommandName ：表示命令名称，比如： copy, cut 等（更多命令见命令）；
aShowDefaultUI：是否展示用户界面，一般情况下都是 false；
aValueArgument：有些命令需要额外的参数，一般用不到；

兼容性

这个方法在之前的兼容性其实是不太好的，但是好在现在已经基本兼容所有主流浏览器了，在移动端也可以使用。
意思就是可以允许运行命令来操作可编辑区域的内容，注意，是可编辑区域。

  1. cut 剪贴当前选中的文字并复制到剪贴板
  2. copy 拷贝当前选中内容到剪贴板
  3. paste 在光标位置粘贴剪贴板的内容，如果有被选中的内容，会被替换

我们java-web工程中复制功能就是借助于clipboard.js插件实现的，使用方法如下：

其实Clipboard插件也是封装了selection和execCommand方法实现的拷贝功能。

3.优缺点比较

Clipboard.js 近几年使用较多，体积相对小，兼容性可以接受，使用还比较方便。
execCommand 新兴势力，safari等主流正在努力兼容。

Clipboard.js 较小，4KB
execCommand是document方法，不用插件直接搞

4.总结

clipboard.js就是封装了一下，本质就是document.execCommand(this.action)。
在项目开发中，java-web已经提供Clipboard.js插件的引用，使用，较为简单清爽；在一些别的活动页面开发时，为了一两个页面中的拷贝功能实现也不需要插件的引用，直接使用document.execCommand方法。

返回一个fulfilled的Promise实例，或原始Promise实例。

  3. 参数为Promise实例，则返回该实例，不做任何修改
  4. 参数为thenable，立刻执行它的.then()

promise学习03-Promise.all

Promise.all()

批量执行一堆函数

  1. Promise.all([p1,p2,p3,…])用于将多个Promise实例，包装成一个新的Promise实例。
  2. 返回的实例就是普通Promise。
  3. 它接收一个数组作为参数
  4. 数组里可以是Promise对象，也可以是别的值，只有Promise会等待状态的改变
  5. 当所有子Promise都完成，该Promise完成，返回值是全部值得数组
  6. 有任何一个失败，该Promise失败，返回值时第一个失败的子Promise的结果。

执行结果：

  1. (1)中传入的是[1,2,3]，不是数组，结果会被立刻返回
  2. (2)中传入的是function，不是Promise实例，也会被立刻执行
  3. （5)中接收到的都是Promise实例，因此会等待1.5s左右返回两个Promise(3)(4)的返回值组成的结果
  4. (8)有p1,p2,p3，同时p2,p3都会返回错误，p2会错误的早一些，所以(9)会被跳过，同时(10)catch触发，捕获到错误，是由第一个触发错误的Promise返回的值(7)

Promise.all() 和.map()使用

Promise.all()最常见的就是和.map()，把一组操作转化成Promise集体的状态。

实现队列

有时候我们不希望所有动作一起发生，而是按照一定顺序，逐个进行。
通过Promise.then()返回一个新的Promise实例的特性，串成队列来进行操作。
如果有一组要做的事，有两种方式：
（1）使用.forEach()、for循环

（2）使用.reduce() 从数组的一端遍历数组，直到另外一端

两个常见错误：

（1）中与forEach连用时，没有把.then()产生的新Promise实例赋给Promise，没有生成队列，造成当Promise完成时，后面所有的then会同时触发
（2）中与reduce连用时，Promise实例创建之后，会立刻运行执行器代码，所以所有的Promise也会立刻执行，不会等待队列的下一步才执行。

promise学习02-then

then的执行效果

  1. .then()接受两个函数作为参数，分别代表fulfilled下的响应函数和rejected下的响应函数
  2. .then()返回一个新的Promise实例，所以它可以链式调用
  3. 当前面的Promise状态改变时，.then()根据其最终状态，选择特定的状态响应函数执行
  4. 状态响应函数可以返回新的Promise，或其他值
  5. 如果返回新的Promise，那么下一级.then()会在新Promise状态改变之后执行
  6. 如果返回其他任何值，则会立刻执行下一级then

1. .then的嵌套

.then()里有.then()的情况
  1. 因为.then()返回的还是Promise实例
  2. 会等里面的.then()执行完，再执行外面的
  3. 对我们来说，此时最好将其展开，会更好读。

上图中因为第一个响应函数里没有return，所以虽然doSomethingElse返回了一个Promise实例，但是这个实例并没有返回给这个then的响应函数，只能看做返回空， 所以下一步会立刻执行。 在第一个then中传入了一个执行函数，前两个的执行时间几乎是一致的，因为是在一个栈中执行的。 因为doSomethingElse返回的是一个Promise实例，而不是一个函数，在Promise规范的定义中，第一个then会被忽略掉，所以finalHandler最后一步执行时，监听的是第一个函数的完成时间。 第二个函数的执行时间是无关紧要的，因为它并不处在队列当中。

3. 错误处理

Promise会自动捕获内部异常，并交给rejected响应函数处理。

只能捕获异步完成回调之后的错误信息，在Promise的执行器中如果发生错误，那么Promise的状态就会被置为rejected，后面fulfilled的响应函数不会被执行，而
后面通过catch来捕获错误的函数就会被执行。

在Promise中错误处理有两种做法：
  1. reject('错误信息').then(null,message=>{})
  2. throw new Errow('错误信息').catch(message =<{})
    推荐使用第二种，更加清晰好读，并且可以捕获前面的错误。
.catch()+.then()会发生什么？

catch也会返回一个Promise实例，并且如果其中没有抛出错误这个Promise实例也是fulfilled的状态。

打开注释调的代码后执行结果：
another err会使得catch(1)返回的Promise实例变为rejected状态，绕过下面两个then函数的执行，进入到下一个catch函数。

注意：建议在所有队列最后都加上.catch(),因为队列的执行全部都是异步的，很可能生成的时候是没问题的，而出问题时却不知道，引发意想不到的问题。
而我们通过捕获，把错误处理一下或记录下来，都会有比较好的效果。
