webpack学习03-核心概念(下)

sourceMap的配置

sourceMap是shong么？

当我们代码写错后，打包运行，在console控制台会有错误提示，但提示的是错误在打包之后的文件位置，而开发阶段，最重要的是错误定位到原文件，

sourceMap是一个映射关系，知道dist/main.js文件出错位置在原文件index.js文件中的位置。

sourceMap使用

在webpack.config.js中配置devtool属性为devtool: 'source-map',而这个是会影响性能
具体查看官网,使用devtool其实就是在使用sourcemap，配置之后，官网表格会显示打包的过程build、rebuild是会变慢的，因为打包过程中要构建这种映射关系.
打包运行后，dist目录下会生成main.js.map文件，其实就是打包后的文件与原文件的映射关系文件。

官网中：cheap、eval、inline、module等都可以自由组合，产生不同的打包效果：
devtool: 'inline-source-map'与devtool: 'source-map'区别，第一种无map文件，map文件会以dataURL的方式直接写在dist/main.js中。main.js.map文件会被变成base64对的字符串放在main.js的底部。
devtool: 'cheap-inline-source-map'与上述的区别，上述会告诉哪一行哪一列即哪个字符发生错误，这是比较耗费性能的，且我们实际只需要知道在哪一行就行，cheap会使打包性能得到一定的提升。
且cheap只对业务代码做映射，对loader等第三方引入的模块文件不做映射，而module会管loader、第三方模块中的错误。
eval是打包速度最快的一种方式，通过eval的执行方式，来生成sourcemap的对应关系，但是针对复杂的代码，该方式的提示可能不够全面。

最佳实践

开发环境mode：development：devtool:'cheap-moudle-eval-source-map'
线上环境mode：production：devtool:'cheap-module-source-map'

sourceMap的原理是什么？

webpack-dev-server不生成dist目录，会放在电脑的内存中，会有效的提升打包的速度，让开发更快，webpackdev-server隐藏的一个目录。

  1. scripts:”webpack-dev-server”命令运行后，打包不会生成dist目录，会帮助我们启动一个localhost服务器。
    1. webpack-dev-server简介：
      是一个小型node.js express服务器
      新建一个开发服务器，可以serve我们pack以后的代码，并且当代码更新的时候自动刷新浏览器
      启动webpack-dev-server后，你在目标文件夹中是看不到编译后的文件的，实时编译后的文件都保存到了内存当中。
    2. 两种自动刷新方式：
      1. iframe mode
        在网页中嵌入了一个 iframe ，将我们自己的应用注入到这个 iframe 当中去，因此每次你修改的文件后，都是这个 iframe 进行了 reload
        命令行：webpack-dev-server，无需–inline
        浏览器访问：http://localhost:8080/webpack-dev-server/index.html
        inline mode
      2. 命令行：webpack-dev-server –inline
        浏览器访问：http://localhost:8080
    3. 安装webpack-dev-server
      npm install webpack-dev-server –save-dev
      可在项目根目录下安装（不加-g，在项目根目录出现node_modules文件夹，内含webpack-dev-server及其依赖包），也可全局安装（加-g 必须sudo）

c. 在webpack.config.js中添加配置
当在一个代码里引入别的模块时，如果希望只更新代码发生变化的模块就要用HMR技术，在webpack.config.js中配置
hot:true;hotOnly:true;以及在plugins中配置相关信息。
