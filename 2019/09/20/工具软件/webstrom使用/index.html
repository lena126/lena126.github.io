  2. compare with the same Repository Version
    与当前版本（的文件）做比较，可以理解为与最新版本比较，也就是可以比较本地工作区和本地版本库的差异
    是指当前文件，与commit后暂存区的区别，比较视图中右侧永远是最新的代码，之间的差别就是你本地修改后未commit的代码
  3. command+d 查看当前版本与历史版本的不同
    当你想要保存当前的暂存区和工作区的状态的时候，你可以使用git stash命令
    这个命令会保存当前的暂存区和工作区的状态，然后返回到HEAD（git reset —hard HEAD）。最新的stach可以在.git/refs/stash中看到。
  • Revert…　　还原代码，要注意与reset的区分，这里仅仅是将本地工作区的代码还原为本地仓库中的最新版本的代码
  • command+option+z revert当前文件
  • checkout　　检出至本地工作区，此时本地已经检出过
  • checkout as new local branch　　检出至本地工作区，并创建新分支
  • compare 　　两个分支进行比较，比如，我想拿当前的test分支与本地的master分支进行比较
  • merge 　　进行合并操作，以比较版本合入当前版本中，merge的过程当中，可能会有冲突（要记住，多人修改同一处代码时，才可出现冲突），稍候介绍怎么处理冲突
  • delete 　　删除当前分支
  • git add：暂存区的目录树被更新，同时工作区修改（或新增）的文件内容被写入到对象库中的一个新的对象中，而该对象的ID 被记录在暂存区的文件索引中。
  • git commit：暂存区的目录树写到版本库（对象库）中，对应的分支会做相应的更新。即HEAD指向的目录树就是提交时暂存区的目录树
  • git pull，将下载（fetch）和合并（merge）放到一个命令里

2. 实用技巧

  • webstrom中command+ - 对当前代码块进行展开/折叠 command+。 实现对当前代码块的折叠展开切换
  • shift + ⬇ ⬆ down with selection
  • option+x 删除光标所在word之前所有字符 control+x 删除当前行光标后内容
  • // todo 代码中注释TODO，command+6 可以查看todolist
  • fn option f12 打开terminal command+。 打开终端设备

3. 开发技巧

1. 看代码是否编译成功，看gitlab中提交记录，pipelines编辑是否成功
2. gitpull只是拉取当前分支的远程代码，与本地分支进行合并
3. git fetch 将origin/master   merge into current才是将远程代码合到当前分支

4.webstrom快捷键

  • command+ 调出设置窗口
  • 双击shift 搜索
  • command+shift+f 搜索（更好用）
  • command+l 跳到指定行
  • command+shift+l 执行pull changes
  • command+shift+k push commits
