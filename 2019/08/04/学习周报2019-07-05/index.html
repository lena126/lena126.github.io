学习周报2019/07/05

1.render概念

React中每个组件有且只有一个render方法，创建组件有class方式创建和函数声明创建两种方式，两种方式返回的有且只有一个顶点节点。render方法的作用就是将react元素渲染到真实的dom中。

2.触发render机制

首先在组件首次加载的时候会执行render，在实例化过程中，当执行componentWillMout之后会执行render，开始将节点挂载在页面上。
其次有两种情况会触发render，一是setState会导致组件的重新渲染，二是接收到新的props属性时。

（1） 状态更新时

（2）props属性改变

只要点击了 App 组件内的 Change name 按钮，就会重新 render。而且可以注意到，不管 Foo 具体实现是什么，Foo 都会被重新渲染。无论组件是继承自 React.Component 的 class 组件还是函数式组件，一旦父容器重新 render，组件的 render 都会再次被调用。

3.render触发优化

当状态更新后，有时只想让相关的组件重新渲染，并不想让不相关的组件也被重新渲染，因此要对其进行相关优化操作。优化方式有多种，以下是常用的两种优化方式：
shouldComponentUpdate(nextProps,nextState)方法中是render函数调用前执行的函数，可以通过nextProps,nextState参数来判断当前场景是否需要重新渲染，当shouldComponentUpdate方法return true则重新渲染，return false则阻止组件渲染。
PureComponent中，只接受props和state参数，如果props和state没有改变，PureComponent不会重渲染，可以一定程度上减少了render带来的消耗。
